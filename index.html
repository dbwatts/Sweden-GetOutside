A Guide to Sweden's Protected Areas

Sweden's Wild Landscapes

This web page presents a map of the country of Sweden, divided into its 25 historical provinces. Sweden’s Provinces, referred to as Landskap in Swedish, have deep historical roots dating back to the medieval era (Högman, 2019). While these provincial boundaries no longer hold administrative or political significance, they remain culturally meaningful to many Swedes and are still frequently referenced in modern conversation (Högman, 2019).

While Sweden’s Landskap are known for their rich cultural heritage, they are further distinguished by awe-inspiring landscapes, each more unique than the last. Sweden is committed to protecting these natural environments, and established it’s first nine National Parks in 1909— the first designated National Parks in all of Europe (Sveriges Nationalparker, 2025). Since that time, Sweden has established 30 national parks and almost 4,000 nature reserves, in an effort to protect and preserve Sweden’s wild spaces and biodiversity (Naturvårdsverket, 2025; Sveriges Nationalparker, 2025).

Sweden’s natural landscape can be characterized by six unique habitat types, including lakes and watercourses, mountains, coast and sea, broad-leaved deciduous forests, wetlands, and coniferous forests (Sveriges Nationalparker, 2025). Each province boasts unique natutral terrain, and the National Parks and Nature Reserves collectively protect all these unique habitats, offering opportunity to experience the diversity of Sweden’s landscape and discover what makes Sweden’s natural environment so special.

This map is a reference guide to Sweden’s vast wilderness, with a directory to learn more about the natural landscapes of this nordic country. The map illustrates all thirty of Sweden’s national parks by province, as well as some of the best-known Nature Reserves in the few provinces in which National Parks have not been established. Learn more about the natural landscapes of each province by clicking on the Landskap name, and click on the park or nature reserve name to learn more about that specific protected area.*

Assignment Objectives

The purpose of this assignment was to enable students, as new developers, to expand our foundational web-development skillset we have developed so far through the GIS: Cartography and Geovisualization program. This assignment enabled not only practice of the skills we have already learned related to HTML, CSS and JavaScript, but to further enhance these skills through the integration of pre-developed tools like ai2html.

Use of the ai2html script enabled further practice in linking and application of JavaScript into our web pages, and also taught us how to transform otherwise static images into dynamic and scalable web maps (or graphics) within our web pages, reinforcing the key concepts of web mapping and responsive design. This assignment also enabled further practice in applying meaningful cartographic customizations, to enable us as developers to further incorporate cartographic design principles and theory to create an engaging and thematically relevant web page. By incorporating hyperlinks to external resources, the web map becomes interactive, enhancing overall user engagement with the web page. Finally, this assignment further reinforced best practices in file organization, which is a key skill required in the development of any mapping or geovisualization project.

Creation of this webpage:

Creation of this responsive web map was made possible through the ai2html script developed by The New York Times Company. To develop this web map, the following steps were taken:

  1. I began by setting up an appropriate folder structure including ai (Adobe Illustrator), img (Image), css (CSS Style) and js (Javascript) folders. I created a new <index.html> file, and a new style.css file in appropriate folders.
  2. Next I developed a basic webpage containing all essential HTML components including a head section (stating the document type declaration, language parameters, character set encoding) and a body container (inclusive of a header, aside, section, and footer containers nested within). I linked my style.css file in the <head> section of my web page, to apply custom styles to the webpage.
  3. To ensure that my webpage would be responsive to expansion and compression of the browser, I needed to incorporate flex parameters to my aside and section containers, which would hold the majority of my web page content. I did this by assigning a class to the main container holding both the “aside”, which would hold my narrative text, and the map “section” that would hold my map. Then, in my style.css file, I called upon this “main-container” class, and applied the “-webkit-flex” parameter. Further, I called upon the aside and on the container that would hold the web map, and assigned flex parameters to each container (aside: 30%, map container: 70%), which would specify that the aside will take up 30% of the viewport and the map container would take up 70% of the viewport. These proportions will dynamically adjust to the user’s screen size.
  4. I also included an “@media” query into the bottom of my style.css file, specifying further parameters to the flex settings. This "@media" query enables the web page to adjust positioning of the containers in the web page, according to the scaling of the browser/viewport. On wide browsers, the containers will appear side-by-side and as expected, with the aside container placed on the left of the map. On very narrow screens or browsers (in this case, less than 800px according to my settings) the containers will dynamically shift, to hold the aside container above the map. In my <style.css> file, I applied the parameter of “column” to the “-webkit-flex-direction” and “flex-direction” settings within my "@media" query.
  5. Now that my web page was largely formatted, I needed to install ai2html on my computer. To do this, I started by copying the JavaScript file provided by The New York Times Company, and pasting it into the js folder that I created within my project folder. This was now saved to my project as “ai2html.js”. To enable ai2html within Adobe Illustrator, I also pasted a copy of the ai2html.js file into the Adobe Illustrator software itself, via the following pathway: “C:\Program Files\Adobe\Adobe Illustrator 2024\Presets\en_US\Scripts. Once I navigated to Adobe’s “Scripts” folder, I placed the ai2html.js file here to successfully install the script and associated functionality into Adobe Illustrator. Note: this pathway may change, depending on the version of Adobe Illustrator held on the machine. Always navigate to the newest version of Adobe Illustrator, to install the script into the version that will be accessed by the user.
  6. Now that the ai2html script had been successfully installed, the next step was to select an SVG file that would be used within the webpage. I downloaded an SVG file of Sweden’s domestic and provincial boundaries, and saved it to my “ai” folder. I then proceeded to open the file in Adobe Illustrator.
  7. From here, I styled the map according to my chosen theme within Adobe Illustrator (i.e. National Parks and Nature Reserves in Sweden). In this case, as the labels on the provinces were not text-based, I removed these label objects and replaced them with text-based labels on the map. This will allow the text to be maintained when brought over to the webpage, preventing the text from being scaled with the map image itself. Note: ai2html functions best when using Arial or Georgia fonts.
  8. Once I was happy with the design of my map, I ran the ai2html script to convert the ai file into HTML code that could be plugged into my existing <index.html> file. This involved first sizing the artboard appropriately to ensure that my map was framed as I wanted it to appear in my web page. I verified that my colour mode was set to RGB (a requirement for ai2html), and saved my document. Next, I ran the ai2html script (File > Scripts > ai2html). Running this script generates a new folder, housed in my project’s “ai” folder called “ai2html-output”. Within this new folder is an HTML file and a default-generated PNG file.
  9. Given that I want to include an SVG file in my web page, to maintain sharp image-quality and dynamic scaling to fit various viewports, I needed to make some revisions to the file and run the script again to adjust the default settings generated in the ai2html-output that would better suit the needs of my web page. The ".ai" file contained a new text box specifying various parameters used in the script. I adjusted the image_format parameter from auto to svg (specifying that the output should be in SVG format). I adjusted the responsiveness parameter from fixed to dynamic to enable the image to expand and shrink along with the browser. Once these parameters were revised, I ran the script again (File > Scripts > ai2html) to generate a new output with the SVG image and a new html file that could be incorporated into my own <index.html> file.
  10. Now that the script was regenerated, I went into the “ai2html-output” folder and moved the output files into the appropriate folders within my project: I moved the html output into the root of my project folder, alongside the <index.html> file, and moved the generated ".svg" file into my img file folder.
  11. Now, I needed to open up the new html file, and copy/paste its content into my <index.html> code. The generated html includes in-line styling to hold the styles developed in Adobe Illustrator. As I have a linked CSS file within my project folder, I copied the script outlining the style properties from the html code and pasted it into my style.css file, below the styling that was already applied prior, but above the "@media query" that was placed at the bottom of the css code. I saved my style.css file.
  12. Next, I copied all of the remaining content from the ai2html output html code, and pasted it into the map-container of my <index.html> file. This would successfully load the styled image of Sweden into my web page, in the appropriate container. From here, I had to adjust some of the html code to ensure that the image would successfully appear on my webpage, noting I had moved the relative location of the ".svg" image. In the generated html code, in the second <div> container that housed the svg image and source, I made the following changes:
    I removed the <style=""> attribute, since the in-line styling had now been transferred into my style.css file. I updated the <alt=""> attribute to include alternative text for the image, and finally, I updated the <src=""> attribute to the relative pathway to my ai2html-generated ".svg" file to reflect its new location in the “img” folder. I did this by copying the relative path of the image, pasting it within the quotation marks of the <src=""> attribute, and finally changing the default back-slash to a forward-slash in the pathway. The result looks like the following: <src="img/Sweden_BackgroundImg-Artboard_1.svg">. I saved my web page and verified that the image was appearing as expected, and was responsive to expansion and compression of the browser.
  13. Finally, to include anchored hyperlinks related to my map’s theme, I sifted through the output html, that was now embedded within my own <index.html> file, and updated each province label with an anchored hyperlink to more information about the National Parks and/or Nature Reserves found in that province. I used <a href="link to web page"> and <target=”_blank”> to anchor the hyperlink and ensure that when clicked, the link will open in a new browser tab.
  14. From there, the last step was to adjust any CSS styling in the linked style.css file as desired to further thematically style my web page.
